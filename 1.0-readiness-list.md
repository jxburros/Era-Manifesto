# Era Manifesto 1.0 Implementation Plan

This document outlines the comprehensive roadmap to transition **Era Manifesto** from its current state to a professional 1.0 production release. The plan integrates essential data safety features, advanced workflow automation, and professional-grade auditing tools.

---

## Phase 1: 1.0 Essentials & Data Safety (The Foundation)
*Focus: Ensuring data integrity, financial transparency, and manual backup capabilities.*

* **CSV Export (Financials & Tasks)**
    * **Goal**: Enable raw data export for external accounting and management.
    * **Technical Steps**:
        * Create `src/utils/csvExport.js` to handle object-to-CSV conversion and browser downloads.
        * Add "Export CSV" buttons to summary and detail tables in `FinancialsView`.
        * Add an export button to `GlobalTasksView` and `TimelineView` that respects current active filters.
* **Hard Backup/Restore UI**
    * **Goal**: Reinforce "User Autonomy" by allowing full manual data ownership.
    * **Technical Steps**:
        * Add a "Generate Backup" button in `SettingsView` to download the state as a `.json` file.
        * Create a file upload handler to validate and ingest backup files into the store.
* **Team Financial Summaries**
    * **Goal**: Aggregate total payables per collaborator.
    * **Technical Steps**:
        * Implement a selector in `TeamView` that filters tasks by `assignedMemberId`.
        * Sum "Effective Costs" using existing precedence logic (Paid > Quoted > Estimated).

---

## Phase 2: Templates, Blueprints & Bundles (Workflow Automation)
*Focus: Transforming the app into a proactive automation engine for release campaigns.*

* **Single Item Templates ("Save as Template")**
    * **Goal**: Allow users to save custom configurations for Songs, Videos, or Releases.
    * **Technical Steps**:
        * Add a "Save as Template" action in item detail views.
        * Create a utility to strip instance-specific data and convert fixed dates into relative offsets.
* **Starter Blueprints & Custom Library**
    * **Goal**: Provide pre-configured defaults (e.g., "Standard Single Rollout") while allowing user-defined templates.
    * **Technical Steps**:
        * Define `src/settings/blueprints.js` for default structures.
        * Implement a `userTemplates` array in `Store.jsx` to store custom user configurations.
* **Campaign Bundles (Multi-Item Templates)**
    * **Goal**: Create a full release package (e.g., 1 Song + 1 Music Video + 1 Digital Release) with a single click.
    * **Technical Steps**:
        * Update `taskEngine.js` to handle recursive generation.
        * Use a single "Anchor Date" (Release Date) to trigger all relative deadlines across the bundle.

---

## Phase 3: Efficiency, Auditing & UI Polish (The Professional Tier)
*Focus: Refining the experience for power users managing large-scale projects.*

* **Batch Actions Bar**
    * **Goal**: Reduce administrative friction by allowing bulk updates.
    * **Technical Steps**:
        * Implement multi-select logic in `GlobalTasksView`.
        * Create a `BatchActionBar` component for bulk updates to status, era, or stage.
* **Activity Log / Change History**
    * **Goal**: Provide a professional audit trail for project modifications.
    * **Technical Steps**:
        * Add an `activityLog` array to `Store.jsx`.
        * Update major store actions to record timestamps, entity IDs, and "before/after" values.
* **Dashboard Customization**
    * **Goal**: Allow users to personalize their "Today" view.
    * **Technical Steps**:
        * Add `dashboardSettings` to `userSettings` in the store.
        * Implement conditional rendering for widgets (Notifications, Progress, Financials) in `TaskDashboardView`.
